OpenCode
AI Best Practices

Context Rotting: when the AI forgets

Limited Context Window
AI agents can only "see" a portion of your project at once. As conversations grow, older context gets pushed out.
The Rot Effect
After many exchanges, the AI forgets earlier files and decisions. It may re-read, make inconsistent changes, or lose track of the architecture.
Prevention
Keep prompts focused. Start fresh sessions for different features. Re-read key files when needed. Use clear, self-contained requests.
Git
Advanced Git

git checkout faceva TUTTO

Poi Git 2.23 (2019) ha separato i compiti in switch (branch) e restore (file)
git checkout (old)
Un unico comando, troppe responsabilità
git checkout <branch> cambia branch
git checkout -b <new> crea e cambia branch
git checkout -- <file> scarta modifiche locali
git checkout <commit> -- <file> recupera file da commit
git checkout <commit> HEAD staccato (detached)
Problema: se branch e file hanno lo stesso nome, Git non sa cosa fare → ambiguità
git switch (new)
Solo branch: chiaro e safe
git switch <branch> cambia branch
git switch -c <new> crea e cambia
git switch --detach <commit> HEAD staccato (esplicito)
Vantaggio: un solo compito → nessuna ambiguità. Se provi a fare switch file, Git rifiuta.
git restore (new)
Solo file: l'ex checkout --
git restore <file> scarta modifiche
git restore --source=<commit> <file> recupera da commit
